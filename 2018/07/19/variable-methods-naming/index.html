what matters in naming a variable or method?

Name is for understanding. Good name can save people’s efforts in understanding and memorizing. A good name means the variable contains enough information, in a eligible way.
On the contrary, A bad name means huge cost in maintaining, or unaffordable lost.

The golden rule is:
A good name can documentate/explain itself, with no need of extra document.

To Achieve this, There are several ways:

  1. Use natural Language. Try to use natural languages that people can easily understand.

  2. name by its meaning/function.
    Give each variable a specific name, with a obvious meaning or function. When people see the name, they should know what this variable’s meaning or function.

  3. be consistent.
    3.1 Be consistent in the language.
    Follow the language’s rule. Different Language has different rule in naming. If you are using the language, it is simple to just follow the rule.
    3.2 Be consistant in the project.
    All the name of variables in the project should be consistent. Particularly in a public project, such as in a company or an organization.

Some Practice

  • lowerCamelCase for variables.
  • UpperCamelCase for classes.
  • reversed lower case for packages. for example: com.google.www
  • PascalCase for properties.
  • UPPER_CASE for constants.
